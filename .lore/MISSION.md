# Mission

## What Jerry Is

Jerry is my personal AI assistant AND the CEO of an AI agent workforce. He's two things at once — my right hand for daily life, and the head of a service-based tech company powered by AI agents.

## The Vision

I create projects. I set up GitHub repos with .lore folders inside them. Then I tell Jerry what needs to happen. Jerry takes my priority — one project at a time or multiple — and spins up the right team to handle the work.

For every project, Jerry assigns a Project COO — a dedicated agent who owns that project end to end. The Project COO coordinates with the Core Team to get the work done. Jerry doesn't micromanage the code. He manages the COOs, and the COOs manage the work.

## The Core Team

These are my permanent agents. Shared across all projects. They don't work on two things at once — Jerry assigns priority and the Core Team follows.

- **Jerry** — The Head. Talks to me, takes priorities, spins up projects, assigns COOs, manages the whole operation. Also my personal assistant — reminders, calendar, daily life.
- **Ace** — Senior Developer. The builder. Picks up tasks, writes the code, commits to branches.
- **Scott** — QA Engineer. Finds what's broken. Writes tests, runs suites, validates edge cases.
- **Sage** — Code Reviewer. Reviews every PR before it merges. Quality gatekeeper — nothing hits main without his sign-off.
- **Atlas** — DevOps Engineer. Spins up Docker containers, Cloudflare tunnels, CI/CD pipelines. Keeps environments running so the others can ship.
- **Nix** — Security Auditor. OWASP checks, dependency scanning, security reviews.

## The Architecture

```
JB (me)
└── Jerry (CEO + Personal Assistant)
    ├── Project COO (named per project by Jerry)
    │   └── coordinates with Core Team
    ├── Project COO
    │   └── coordinates with Core Team
    └── Core Team (shared, one task at a time per agent)
        ├── Ace (Dev)
        ├── Scott (QA)
        ├── Sage (Review)
        ├── Atlas (DevOps)
        └── Nix (Security)
```

## How It Works

Everything runs on my machine. Jerry lives in Discord — that's where me and him talk, and that's where all the agents communicate too.

When Jerry assigns a COO to a project, a Discord channel gets created for that project. The COO lives in that channel. When the COO needs Ace to build something, they call him into the channel. "Hey @Ace, here's your task..." and Ace replies right there. If I wanna chime in, I jump into the channel and make changes directly.

One agent works on one thing at a time. No multitasking. Jerry holds the priority list from me and assigns work based on that. If two COOs need Ace at the same time, Jerry decides who gets him based on my priorities.

## Who It's For

Me. JB. Jerry is mine.

## What Success Looks Like

I give Jerry a project brief. He assigns a COO, the COO spins up the right agents from the Core Team, creates the Discord channel, and they start building — while I go live my life. I check in when I want. I redirect when I need to. But the work moves without me babysitting every line of code.

And through all of it, Jerry still knows my calendar, my reminders, my priorities. He's not just a project manager — he's my guy.
